## üöÄ How to Upload to Brevo

### Method 1: Using Brevo's Email Template Builder

1. **Log in to Brevo**
   - Go to https://app.brevo.com
   - Navigate to `Campaigns` ‚Üí `Email Templates`

2. **Create New Template**
   - Click `Create a new template`
   - Select `Rich text editor`

3. **Import HTML**
   - Click on the `</>` (code) icon in the editor
   - Copy the entire HTML content from one of the template files
   - Paste it into the code editor
   - Click `Save`

4. **Configure Variables**
   - Brevo automatically detects `{{variable}}` syntax
   - Test variables using the preview feature
   - Map variables to contact attributes or set default values

5. **Save Template**
   - Give it a descriptive name (e.g., "Brandora - Welcome Email")
   - Add it to a category for organization

### Method 2: Using Brevo's Drag & Drop Editor

1. Create a new template with the drag & drop editor
2. Add a "Code" block
3. Paste the HTML template into the code block
4. Save and preview

### Method 3: Via API (For Automated Deployment)

```bash
curl -X POST "https://api.brevo.com/v3/smtp/templates" \
  -H "accept: application/json" \
  -H "api-key: YOUR_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "tag": "Brandora",
    "sender": {
      "name": "Brandora",
      "email": "hello@brandora.com"
    },
    "templateName": "Welcome Email",
    "htmlContent": "PASTE_HTML_HERE",
    "subject": "Welcome to Brandora! Let'\''s Build Your Brand üéØ"
  }'
```

## üß™ Testing Instructions

### Pre-Upload Testing

1. **HTML Validation**
   - Use https://validator.w3.org to validate HTML
   - Check for any syntax errors

2. **Email Client Testing**
   - Use https://litmus.com or https://emailonacid.com
   - Test across Gmail, Outlook, Apple Mail, etc.

3. **Mobile Responsiveness**
   - Test on actual devices or use browser dev tools
   - All templates are designed to be mobile-responsive

4. **Inline CSS Check**
   - All CSS is inlined (no external stylesheets)
   - Safe for email clients that strip `<style>` tags

2. **Variable Testing**
   - Create test contacts with sample data
   - Send actual test campaigns to verify variable replacement

### Adding Your Logo

Replace the text "Brandora" header with an image:

```html
<!-- Replace this -->
<h1 style="margin: 0; font-family: 'Montserrat', Arial, sans-serif; font-size: 28px; font-weight: 700; color: #070058;">
    Brandora
</h1>

<!-- With this -->
<img src="https://yourdomain.com/logo.png" alt="Brandora" width="180" style="max-width: 180px; height: auto;">
```

### Modifying Fonts

The templates use Google Fonts (Montserrat for headings, Poppins for body) with fallbacks:
- To change fonts, update the `font-family` properties
- Always include web-safe fallbacks (Arial, Helvetica, sans-serif)

### Adding UTM Parameters

Add tracking parameters to your URLs:

```
{{upgrade_url}}?utm_source=brevo&utm_medium=email&utm_campaign=day3_upgrade
```

## ‚úÖ Best Practices

1. **Subject Lines**
   - Keep under 50 characters for mobile
   - Include personalization when possible
   - Test A/B variations

2. **Preview Text**
   - Each template includes preview text
   - Customize to match your message
   - Keep under 100 characters

3. **Send Times**
   - Test different send times for your audience
   - Generally, Tuesday-Thursday 10 AM-2 PM performs well
   - Avoid weekends for business emails

4. **Frequency**
   - Don't overwhelm subscribers
   - Space upgrade emails at least 3-4 days apart
   - Monitor unsubscribe rates

5. **Personalization**
   - Always use first names when available
   - Segment based on assessment scores
   - Customize recommendations based on user data

## üêõ Troubleshooting

### Issue: Variables not replacing
- **Solution**: Ensure variable names match exactly (case-sensitive)
- Check that variables are mapped in Brevo contact attributes

### Issue: Images not displaying
- **Solution**: Use absolute URLs (not relative paths)
- Ensure images are hosted on HTTPS

### Issue: Layout broken in Outlook
- **Solution**: Templates use table-based layouts (Outlook-compatible)
- Test specifically in Outlook if issues persist

### Issue: Mobile rendering problems
- **Solution**: Test with actual devices
- Use Brevo's mobile preview feature

### Issue: Emails going to spam
- **Solution**:
  - Ensure SPF, DKIM, and DMARC are configured
  - Keep good text-to-image ratio (templates follow this)
  - Avoid spam trigger words
  - Warm up your sending domain
