# Icon Library & Usage Guide

## Icon Style

### Design Principles

**Visual Style:**
- **Type:** Outline/stroke-based icons
- **Weight:** 2px stroke weight at base size (24×24)
- **Corners:** Rounded corners (2px radius)
- **Grid:** 24×24 pixel grid
- **Style:** Minimal, geometric, consistent

**Why Outline Style:**
- Clean and modern
- Scalable and flexible
- Works on any background
- Pairs well with typography
- Accessible and clear

---

## Technical Specifications

### Base Grid System
- **Canvas:** 24×24 pixels
- **Live area:** 20×20 pixels (2px padding)
- **Stroke weight:** 2px
- **Corner radius:** 2px
- **Cap style:** Rounded
- **Join style:** Rounded

### Size Variations

#### Small Icons
- **Size:** 16×16 pixels
- **Stroke:** 1.5px
- **Usage:** Inline with text, tight spaces, mobile

#### Medium Icons (Standard)
- **Size:** 24×24 pixels
- **Stroke:** 2px
- **Usage:** UI elements, buttons, navigation

#### Large Icons
- **Size:** 32×32 pixels
- **Stroke:** 2.5px
- **Usage:** Feature callouts, empty states

#### Extra Large Icons
- **Size:** 48×48+ pixels
- **Stroke:** 3px
- **Usage:** Hero sections, illustrations, marketing

## Usage Guidelines

### Do's

✓ **Use icons to:**
- Enhance recognition and scannability
- Support text labels (not replace them)
- Indicate actions and states
- Create visual hierarchy
- Improve navigation
- Represent concepts visually

✓ **Maintain consistency:**
- Same stroke weight within a view
- Same size for similar importance
- Same color coding for states
- Aligned to grid/baseline

✓ **Ensure accessibility:**
- Include aria-labels for screen readers
- Don't rely on color alone
- Provide text alternatives
- Maintain sufficient contrast

---

### Don'ts

✗ **Don't:**
- Use icons without context or labels (except universally recognized ones)
- Mix icon styles (outline + filled)
- Use too many icons in one view
- Make icons the only indicator
- Use decorative icons excessively
- Stretch or distort icons
- Use low-contrast colors
- Create custom icons without following style guide

---

## Icon + Text Combinations

### Horizontal Layout
Most common for buttons and navigation.

```
[Icon] Label
```

**Spacing:** 8px between icon and text
**Alignment:** Vertically centered
**Example:** [+] Add Assessment

---

### Vertical Layout
Used in feature grids and cards.

```
[Icon]
Label
```

**Spacing:** 12px between icon and text
**Alignment:** Center-aligned
**Example:**
```
  [Target Icon]
  Positioning
```

---

### Icon Only
Only for universally recognized icons.

**Acceptable:**
- Search (magnifying glass)
- Menu (hamburger)
- Close (X)
- Settings (gear)
- Profile (user)

**Provide:**
- Tooltip on hover
- Aria-label for screen readers
- Sufficient click/tap target (minimum 44×44px)

## File Formats and Export

### SVG (Primary Format)
- **Usage:** Web, apps, print (when possible)
- **Benefits:** Scalable, small file size, style-able with CSS
- **Naming:** kebab-case.svg (e.g., `arrow-right.svg`)

### PNG (Raster Format)
- **Usage:** When SVG isn't supported
- **Sizes:** Export at 1x, 2x, 3x for different screen densities
- **Naming:** `icon-name@1x.png`, `icon-name@2x.png`, `icon-name@3x.png`

### Icon Font (Optional)
- **Usage:** Legacy projects
- **Benefits:** Easy color changes, one HTTP request
- **Drawbacks:** Less flexible than SVG, accessibility concerns

## Implementation

### HTML/SVG
```html
<svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
  <path d="M12 2L2 7l10 5 10-5-10-5z"/>
  <path d="M2 17l10 5 10-5"/>
  <path d="M2 12l10 5 10-5"/>

```

```css
.icon {
  width: 24px;

  stroke: currentColor;
  stroke-width: 2px;
  stroke-linecap: round;
  stroke-linejoin: round;
  fill: none;
```

## Accessibility Guidelines

### Screen Readers

**Decorative icons (no aria-label needed):**
```html
<svg aria-hidden="true">...</svg>
```

**Functional icons (provide aria-label):**
```html
<button>
  <svg aria-label="Close menu">...</svg>
</button>
```

**Icon + visible text (hide icon from screen reader):**
```html
<button>
  <svg aria-hidden="true">...</svg>
  Close
</button>
```

---

### Color Contrast

**Icon-only:**
- Minimum 3:1 contrast ratio against background
- Larger icons (24px+) should meet this easily

**Icon + text:**
- Follow text contrast requirements (4.5:1 for small text)

---

### Touch Targets

**Minimum touch target:** 44×44 pixels (mobile)
**Icon smaller than 44px?** Add padding to button/link

```css
.icon-button {
  padding: 12px; /* Makes 24px icon have 48px total target */
  min-width: 44px;
  min-height: 44px;
}
```

## Icon Naming Conventions

**Format:** `category-name.svg`

**Examples:**
- `arrow-right.svg` (not `rightarrow.svg`)
- `user-plus.svg` (not `adduser.svg`)
- `chart-bar.svg` (not `barchart.svg`)
- `file-text.svg` (not `document.svg`)

**Rules:**
- Use kebab-case (lowercase, hyphens)
- Be descriptive and specific
- Use common/expected names
- Avoid abbreviations
- Include orientation if relevant (arrow-left vs arrow-right)

## Customization Guidelines

### When You Can Customize

**Allowed:**
- Change color (using currentColor or CSS)
- Change size (proportionally)
- Change stroke width (within 1.5px - 3px range)
- Apply opacity
- Add animations (rotation, fade)

### When You Can't Customize

**Not allowed:**
- Change stroke style (always rounded)
- Add fills (keep outline style)
- Distort proportions
- Add gradients or effects
- Modify paths significantly
- Mix with other icon styles

---

## Creating New Icons

If you need an icon that doesn't exist in the library:

1. **Check the library first** - might exist under different name
2. **Follow the grid** - 24×24 canvas, 2px padding
3. **Match the style** - 2px stroke, rounded caps/joins
4. **Test at different sizes** - should be clear at 16px
5. **Keep it simple** - avoid excessive detail
6. **Get approval** - from design team before adding
7. **Document it** - add to appropriate category
8. **Export properly** - SVG + PNG at multiple sizes
